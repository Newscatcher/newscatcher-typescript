## 2.0.0 - 2026-03-16
* The `AggregationGetRequest` and `AuthorsGetRequest` interfaces have been removed, along with their associated enum types `AggregationGetRequestPublishedDatePrecision`, `AggregationGetRequestSortBy`, and `AuthorsGetRequestPublishedDatePrecision`, `AuthorsGetRequestSortBy`. Existing code referencing these types will need to be updated to use alternative request interfaces or methods.
* The SearchGetRequest interface and related search types have been removed. Applications using these types will need to be updated to use the current search API interfaces.
* The search similar functionality has been removed from the SDK. The `SearchsimilarClient` class and all related types are no longer available. If your application uses `client.searchsimilar.get()` or `client.searchsimilar.post()` methods, you will need to remove these calls or find alternative approaches for similar article functionality.
* Breaking changes to client structure: The `aggregation` client has been renamed to `aggregationCount`, the `searchsimilar` client has been removed entirely, and several client properties have been renamed (`latestheadlines` → `latestHeadlines`, `searchLink` → `searchByLink`). Update your code to use the new client names and remove any usage of search similar functionality.
* Breaking API restructuring with improved naming conventions. All response types now use Get/Post prefixes (e.g., `AggregationGetResponse` is now `GetAggregationCountResponse`). All request types follow the same pattern (e.g., `AuthorsPostRequest` is now `PostAuthorsRequest`). Client methods now use consistent `get()` and `post()` naming instead of resource-specific method names (e.g., `client.breakingNews.breakingNewsGet()` is now `client.breakingNews.get()`). Resource paths have been updated (`latestheadlines` → `latestHeadlines`, `searchLink` → `searchByLink`). The `NotTheme` type no longer accepts arrays - use comma-separated strings instead. The `PublishedDatePrecision` enum has been changed to a string type.
* The Theme type now only accepts string values instead of string arrays. If you were passing arrays to theme parameters, convert them to comma-separated strings. Several API request and response types have been renamed for consistency (e.g., AuthorsGetRequest is now GetAuthorsRequest). The SDK now supports selective imports of individual API modules and adds new parameters for link text filtering, clustering, and sentiment analysis.
* Several response fields that were previously optional are now required, improving type safety but potentially breaking existing code. Fields like `title`, `link`, `domain_url`, `id`, and `score` in `ArticleEntity`, and pagination fields like `status`, `total_hits`, `page`, `total_pages`, and `page_size` in `BaseSearchResponseDto` are now guaranteed to be present. Code that previously handled these fields as potentially undefined will need to be updated.
* New request type definitions available for aggregation count, authors, breaking news, and latest headlines endpoints. These TypeScript interfaces provide complete type safety and documentation for all search parameters and configuration options.
* New request interfaces available for search operations including GetSearchRequest, GetSearchByLinkRequest, and GetSourcesRequest with comprehensive filtering options. The SDK now includes a makePassthroughRequest function for making custom HTTP requests while leveraging the SDK's authentication, retry logic, and logging configuration.
